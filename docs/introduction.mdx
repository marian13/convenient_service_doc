Yet another approach to revisit the [service object](https://www.toptal.com/ruby-on-rails/decoupling-rails-components) pattern in [Ruby](https://www.ruby-lang.org/en/), but this time focusing on the unique, [opinionated](https://rubyonrails.org/doctrine#omakase) features.

## WARNING ❗❗❗

This library is under heavy development. Public API may be subject to change. The first major release is still to come. Use the current version at your own risk. Thanks.

## Features

### Common Features

Common features (with slight variations) compared to existing [alternatives](https://github.com/markets/awesome-ruby#business-logic).

- [JSend](https://github.com/omniti-labs/jsend) and [dry-monads](https://dry-rb.org/gems/dry-monads/1.3/result/) inspired Results.

- Steps similar to [Trailblazer Operation Wiring](https://trailblazer.to/2.1/docs/operation#operation-overview-wiring).

- [Active Model Attributes](https://api.rubyonrails.org/classes/ActiveRecord/Attributes/ClassMethods.html#method-i-attribute) integration influenced by [ActiveAttr](https://github.com/cgriego/active_attr#attributes) (optional).

- [Active Model Validations](https://guides.rubyonrails.org/active_model_basics.html#validations) integration like in [ActiveInteraction](https://github.com/AaronLasseigne/active_interaction#validations) (optional).

### Unique Features

Features specific to this gem.

- Almost full customization.
