Yet another approach to revisit the [service object](https://www.toptal.com/ruby-on-rails/decoupling-rails-components) pattern in [Ruby](https://www.ruby-lang.org/en/), but this time focusing on the <span data-tip="Unique features have special care and the highest priority">unique</span>, <span data-tip="The initial stack of features is completely predefined">opinionated</span>, <span data-tip="It is assumed that you agree with the most of the choices">moderately obtrusive</span>, but <span data-tip="But you still have a way to customize almost everything to your own preferences">not mandatory</span> features.

## ❗❗❗ WARNING ❗❗❗

This library is under heavy development. Public API may be subject to change. The first major release is still to come. Use the current version at your own risk. Thanks.

## Features

### Common Features

Common features (with slight variations) compared to existing [alternatives](https://github.com/markets/awesome-ruby#business-logic).

- [dry-monads](https://dry-rb.org/gems/dry-monads/1.3/result/) inspired Results.

- Steps similar to [Trailblazer Operation Wiring](https://trailblazer.to/2.1/docs/operation#operation-overview-wiring) to organize [Railway Oriented Programming](https://blog.logrocket.com/what-is-railway-oriented-programming/).

- Quick constructor generation by [dry-initializer](https://dry-rb.org/gems/dry-initializer/3.0/) (optional).

- Attributes auto-casting powered by [Active Model Attributes](https://api.rubyonrails.org/classes/ActiveRecord/Attributes/ClassMethods.html#method-i-attribute) and influenced by [ActiveAttr](https://github.com/cgriego/active_attr#attributes) (optional).

- Input parameters validations by [Active Model Validations](https://guides.rubyonrails.org/active_model_basics.html#validations) just like in [ActiveInteraction](https://github.com/AaronLasseigne/active_interaction#validations) (optional).

### Unique Features

Features specific to this gem.

- [The menu is omakase](https://rubyonrails.org/doctrine#omakase).

- [Urge](https://www.vocabulary.com/dictionary/urge) to check Result state to mitigate [The Billion Dollar Mistake](https://www.infoq.com/presentations/Null-References-The-Billion-Dollar-Mistake-Tony-Hoare/) delighted by [Null Reference Checks in Crystal](https://crystal-lang.org/) and [Null Safety in Kotlin](https://kotlinlang.org/docs/null-safety.html).

- Simple, predictable, and powerful at the same time [plugin-based customization](https://roda.jeremyevans.net/documentation.html).

- [One-time delayed](https://manny.codes/3-practical-uses-of-ruby-method-missing/#building-dsls-and-libraries-with-method_missing) [Concerns](https://api.rubyonrails.org/classes/ActiveSupport/Concern.html) and [Middlewares](https://github.com/Ibsciss/ruby-middleware) in the core.
